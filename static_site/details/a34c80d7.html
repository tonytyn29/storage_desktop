双AI精选三股

ID: a34c80d7 | 类型: 多因子选股 + 深度学习动量 | 来源: b2324_56.双人工智能配合，样本外夏普3.9.txt | 分析日期: 2026-01-11 18:21:49

策略逻辑概要

先用16个技术因子的 XGBoost 模型把全市场股票筛到“未来20日大概率上涨”的 10% 左右样本；再用 60 日 K 线序列输入的三套 CNN-LSTM 深度模型（投票/求和）对上述样本二次打分，选出得分最高的 3 只小市值且盈利为正的股票等权买入，每周一 9:30 调仓；同时对前一交易日涨停的持仓做尾盘（14:00）动态止盈管理。

股票池

A 股除创业板/科创板/北交所（代码 3/4/8/68 开头）、ST、停牌、未上市满 180 个交易日、新股涨跌停开盘、财务亏损、EPS≤0，以及昨日涨停但今日已开板的个股。

买入信号

XGBoost 预测概率 >0.5 且位于全部股票得分 Top10%, 深度模型综合得分排名前 3

卖出信号

周一调仓时不再出现在最新 Top-List 且不是昨日涨停封板股, 昨日涨停股尾盘 14:00 打开涨停板（即时卖出）

仓位管理

目标持仓 3 只股票，等权分配（现金/待买数量动态平分）。未设置总仓位上限或跌幅止损；策略自身通过严格选股与小仓分散控制风险。

交易频率

核心调仓：run_weekly(… , 1, '9:30') → 每周一 9:30；附加监控：每日 14:00 检查涨停股，14:50 若全天禁止交易则强平。

手动执行指南

盘前：准备因子与模型无须人工；9:25-9:30 根据选股列表挂当日涨停价委托买入；14:00 留意昨日涨停股是否开板并择机卖出；14:50 若全天停牌或异常需手动清仓。

代码审计详情

所有因子与 attribute_history 均以 context.previous_date 及 count 参数截断；未见 history/pre-open 取未来价；仅 use_real_price=True 与 0 滑点假设带来偏乐观而非未来数据。

灰色区域警告

真实交易无法 0 滑点, attribute_history 默认填补停牌数据可能与实盘有差异, 模型文件可能在训练阶段使用过未来数据，但代码层面无法验证

执行分析

小资金 (<500 万) 可人工+半自动执行；大资金流动性成为瓶颈。

10万资金场景

几乎无冲击，人工挂单即可。

100万资金场景

需拆单+盯盘，但可完成。

1000万资金场景

单票平均 300 万以上，对小市值股深度明显，滑点与冲击不可忽略。

1亿资金场景

难以在小市值股票中布仓且不触及 5% 持股红线，策略失效。

大资金不友好原因

核心聚焦小流通市值股票，容量有限。

行为偏差利用原因

部分收益来源于涨停延续+动量，但整体依赖模型判分，非纯情绪套利。

跨阶段稳健性原因

机器学习模型对市场结构变动敏感；周级频率有一定稳健性但须滚动训练。

最差环境

流动性枯竭/连续跌停, 企业盈利快速下修导致 EPS 失真

可验证预测

样本外一年内组合夏普 >2, 组合月度换手率约 400%, 持仓平均流通市值 < 50 亿

一句话总结

策略结构清晰、未来函数风险低，但执行与容量主要受限于小市值和 0 滑点假设。

优化建议

1) 引入滑点与真实手续费；2) 扩大持股数 + 动态仓位；3) 定期重训练模型并引入财报发布日期约束。

假设条件

未给出资金规模—以上执行与容量评分假设 100-5000 万人民币区间, 手续费、滑点模型缺失—默认聚宽成本参数并视为用户后续修正
