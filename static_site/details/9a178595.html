筹码分布多因子Transformer

ID: 9a178595 | 类型: 多因子选股 | 来源: b2324_77.【复现】筹码分布因子.txt | 分析日期: 2026-01-11 18:28:19

策略逻辑概要

用 60 日内的成交均价及换手率推算市场筹码的获利/亏损分布（ARC、VRC、SRC、KRC 等）与经典陈浩筹码指标（CYQK_C、ASR、CKDW、PRP 等），刻画投资者因为前景理论产生的处置效应。当大部分筹码处于浮盈且集中时，抛压小、后续上涨概率大；若浮亏集中则反弹概率大。以 Transformer 模型学习这些因子与未来 1 日收益的非线性关系，对全市场 / 沪深 300 股票每日打分，买入高分、卖出低分，从而赚取由行为偏差导致的定价误差。

股票池

两套实验：① dataset_config("pool") → 默认 Qlib A 股去掉 ST/停牌 5 日以上股票；② dataset_config("csi300") → 沪深 300 成分。其余过滤条件未在代码体现。

买入信号

当日收盘后，Transformer 预测的 next day return 排名前 TopK 的股票

卖出信号

不再位于 TopK, 达到持仓最大回溯期（Qlib 默认每日完全换仓）

仓位管理

Qlib Backtest 默认等权重、单只≤2%~3%（top50 时），总仓位 100%；未见额外风控。

交易频率

日级调仓；在 after_close 回调做预测，次日开盘下单（Qlib PortfolioSimulator 默认行为）。

手动执行指南

盘后跑脚本生成次日买卖列表；开盘集合竞价一次性下单即可。若用聚宽，放在 before_open 计算、开盘委托。

代码审计详情

全部因子仅用到历史行情；LABEL0(未来收益)只在模型训练/评估阶段使用，没有混入特征。未发现 peek 的函数调用；但自写 clean_factor_data、get_factor_group_returns 未展示源码，无法 100% 排除。

灰色区域警告

Dataset prepare(raw) 会把当日收盘价作为特征，实盘时只能在收盘后使用。, 未对停牌/涨跌停可交易性做进一步过滤。

执行分析

小～中等资金可落地；大资金受仓位上限与冲击成本限制。

10万资金场景

几乎无冲击，完全可执行。

100万资金场景

仍可一次性成交。

1000万资金场景

需关注全市场版本中的小盘股成交量，可能拆单。

1亿资金场景

大幅超买卖盘，除非限制在沪深 300 或扩大持仓数。

大资金不友好原因

因子在小盘股/换手高票中更显著；容量有限。

行为偏差利用原因

核心利用处置效应等行为偏差构建因子，符合理论。

跨阶段稳健性原因

训练 2014-17，验证 18-20，测试 21-23 表现仍佳，但缺少 08~13、疫情后完整检验。

最差环境

极端单边行情导致筹码完全右移/左移，因子失效, 监管导致量化资金受限、成交成本骤增

可验证预测

同一日 ARC>0&VRC小的股票未来 1~5 日超额收益为正, SRC<0 时反弹概率高, 综合模型 Top20 组合半年跟踪年化超额>8%

一句话总结

筹码分布+深度模型思路合理，日级调仓可行，但需解决小盘流动性与可能的成分前视。

优化建议

1) 加入成交量过滤与交易成本约束；2) 用 rolling expanding retrain 保持模型新鲜；3) 明确历史成分表避免前视。

假设条件

使用 Qlib Backtest 默认参数：topk=50、等权、手续费双边 0.1%、无滑点模型, 次日开盘可完全按昨晚信号成交；未考虑涨跌停、停牌、限价未成交等 A 股特殊约束
