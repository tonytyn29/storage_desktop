二板排板动量打板

ID: a175fa31 | 类型: 事件驱动 / 情绪动量 | 来源: b2324_9.10倍-二板排板战法优化-无未来函数.txt | 分析日期: 2026-01-11 18:32:32

策略逻辑概要

利用“首板—次日拉二板”这一 A 股日内情绪链条：昨日出现第一个涨停（且非一字板、非创业板）的股票，如果今日早盘 9:33 以后出现多次触及涨停并放量但尚未封死，说明资金正尝试冲击第二个涨停。策略在此时用全部可用现金冲入，博取午后继续冲板或情绪抬升的超额收益；若下午未能封板且出现跌破当日高点 3% 或其它弱势形态，则尾盘止盈/止损离场。

股票池

全部 A 股 → 剔除 ST → 剔除创业板(300开头) → 仅保留前一交易日首板涨停且前一日前未涨停的股票 → 通过 20/60 日波动率、假涨停数等筛选“平整”个股。

买入信号

今日 9:33 以后，分钟级 close==high_limit 的次数 >3（多次摸板）, 当前价 > 昨收 *1.07 且 < 当日涨停价；且今日开盘价 < 涨停价*0.98（非一字板）

卖出信号

14:40 以后：当前价 < 当日最高价*0.97（高点回落 3%）且 > 跌停价, 14:40 以后：当前价 < 持仓成本*0.93 且 < 当日开盘价, 10:30～14:40：当前价 < 昨收*1.03（冲板失败）

仓位管理

每次开仓将全部可用资金买入单只标的（order_value），无分散；无总仓位/回撤控制；手续费仅按万 3 佣金+千 1 印花税；未设置滑点/停牌处理。

交易频率

选股每日盘前；下单在 9:33 以后分钟级扫描，通常 0~1 次买入；卖出最早 10:30，最晚 14:55 前；平均持有 <1 天，极短线。

手动执行指南

1) 盘前（09:20 前）根据昨日首板名单+平整条件生成候选池；2) 09:33 以后盯盘，若触发买入条件立即下单；3) 下午 14:40 以后动态根据价格触发卖出；4) 未成交的候选股 14:45 后放弃；5) 每日收盘清空 help_stock。

代码审计详情

所有 get_price / get_current_data 均以 context.current_dt 或昨日为 end_date，未引用未来数据；未见 backtest.set_price 调用。

灰色区域警告

minute 查询使用 start_date 固定 09:31，不同交易所可能开盘延迟导致缺口。, 大量使用 get_price 在盘中循环调用，实盘易触发频控。, 未做停牌/退市校验，可能出现无法成交或强制平仓。

执行分析

小资金可人工打板；量化框架需高频接口、撤单逻辑；大资金无容量。

10万资金场景

可手动/半自动冲板，回车即成交，容量够用。

100万资金场景

需拆单，部分票封单量不足，滑点抬升显著。

1000万资金场景

大部分时间无法吃到足够筹码，策略失效。

1亿资金场景

几乎不可执行，排队单量受限且监管风控高。

大资金不友好原因

依赖涨停板窄价位，封单量通常几百万到几千万，无法容纳大资金。

行为偏差利用原因

核心收益来自情绪溢价和涨停板博弈，典型行为金融场景。

跨阶段稳健性原因

涨停板情绪在牛市/情绪市才活跃，熊市大幅回撤；参数针对特定阶段。

最差环境

系统性熊市或无连板行情, T+0/涨跌幅限制放宽导致板块套利结构改变

可验证预测

在情绪高涨的阶段（日涨停家数 >80）策略收益显著高于基准, 持仓分布呈现显著小市值、低流通特征, 当板块龙头封板失败时，组合当日收益大幅回撤

一句话总结

典型短线打板脚本，未来函数风险低，但成交和滑点问题严重，仅适合小额半自动操作。

优化建议

引入盘口排队逻辑、分散持仓、加日内撤单/追单保护，增加风控（跌停保护、隔夜停牌风险控制）。

假设条件

未提供初始资金，默认聚宽 100w；不影响逻辑评估。, 未提供自定义滑点模型，默认 0 滑点；分析中已假设真实会有较大正滑点。, 默认全部买卖为市价单(order_value/order_target)且可立即成交；实盘需排队，可能部分成交。
