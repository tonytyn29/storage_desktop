龙头首阴打板策略

ID: b19d26e3 | 类型: 事件驱动 / 情绪动量 | 来源: b2324_10.龙头首阴战法改版二.txt | 分析日期: 2026-01-11 14:43:34

策略逻辑概要

专找近期出现多次涨停且疑似成为‘龙头股’的标的，在其第一次出现“阴线或换手但仍强势”的次日于盘中追涨（接力）。逻辑依赖短期资金情绪：连板龙头往往在第一次调整后继续上攻，利用第二天资金回流获得超额收益；若走势不及预期则在当日下午或盘中止盈/止损。

股票池

初始用 get_all_securities() 得到全市场股票 → 剔除创业板 300*** 与 ST → 剔除上市不足 1080 日个股 → 进一步筛选前一交易日（pre_date）涨停的股票。

买入信号

信号1：昨日（pre_date）未涨停，且昨日之前有≥2个有效连板；今早日线满足：开盘>昨收*1.07，当前价>昨高限*0.99 且 >昨收*1.03，且未触及今日涨停。满足时全仓买入；, 信号2：09:40-11:20 之间若当前价>今日开盘且>昨收*1.03，其他条件同上；或昨日封板且当前价>昨高限*1.06 时买入。

卖出信号

14:40 以后：当前价 < 当日最高价*0.869；或浮亏>8%；或浮盈<20%且出现明显回落；或炸板/高位放量等复杂组合条件。, 14:40 以前：对连续涨停≥12板、浮亏8%、开盘即封死涨停后被砸等情形做提前止盈/止损。

仓位管理

每次 order_value(stock, cash) ——即把全部可用资金买入单只股票；无分散。风控仅体现在多条卖出规则及当日 available_cash>2 万的限制。

交易频率

理论上日内 0-1 次买入、1 次卖出；触发条件严格依赖分钟数据。下单发生于 run_daily(market_open, time='every_bar')，即每分钟均可能下单。

手动执行指南

盘前：根据 yesterday 数据跑选股脚本，得出候选列表；盘中：09:30-11:20 关注候选股并按规则追涨；14:40 再根据规则检查是否卖出；收盘后清空列表，复盘并记录次日候选。

代码审计详情

所有 get_price end_date 均 ≤ context.current_dt 或 ≤ 前一交易日；未发现使用未来价格。但使用 get_all_securities() 默认当前日期构成‘存活者偏差’——非典型未来函数。

灰色区域警告

选股时使用今日存量股票池可能引入偏差。, minute 级策略需确认回测与实盘撮合一致性。, 单票满仓+小票打板回测易高估成交率。

执行分析

小资金 (<100 万) 依靠市价追击尚可；资金稍大则因涨停板排队、冲击成本造成执行困难。

10万资金场景

大概率可在一两个档位内成交；滑点~0.5-1%。

100万资金场景

排队时间增加，当天可能买不满或卖不掉，滑点>2%。

1000万资金场景

多数标的小于 50 亿市值且涨停封单有限，基本无法一次性成交。

1亿资金场景

同上，接近不可执行。

大资金不友好原因

靠涨停板接力，小票容量低；封单本就拥挤。

行为偏差利用原因

利用市场情绪、涨停效应与龙头崇拜等行为偏差获取超额收益。

跨阶段稳健性原因

强依赖牛市/情绪高涨阶段；熊市连板极少时失效。

最差环境

市场无题材热点、连板稀缺, 监管升级导致涨停松动或加强异动停牌

可验证预测

策略收益高度集中于连板高发年份（月）, 在跌停连板时几乎无交易信号, 回撤主要来自高位炸板当日 >8% 下跌

一句话总结

典型打板接力脚本：思路清晰但执行与容量受限，适合小资金高风险偏好者。

优化建议

引入成交量/流通市值过滤；分仓控制；改用 VWAP 或集合竞价排队提高成交率。

假设条件

未提供初始资金，默认 JoinQuant 回测 100 万起；所有 order_value 以 available_cash 为准。, 使用聚宽默认撮合（逐笔成交、无滑点）和手续费模型；真实环境需考虑排队与涨跌停无法成交。, 未设置 slippage_model，分析中以情景假设给出滑点判断。
